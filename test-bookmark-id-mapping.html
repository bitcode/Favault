üéØ Fix Applied: Added Data Attributes to BookmarkItem.svelte

The root cause was that BookmarkItem components weren't setting the necessary data attributes that the enhanced drag-drop system needs to identify bookmarks.

‚úÖ After (Fixed)

Test 1: Verify Data Attributes Are Set

  1. Load the FaVault extension in Chrome
  2. Open Chrome DevTools (F12)
  3. Go to the Elements tab
  4. Find a bookmark element with class "bookmark-item"
  5. Check that it has the following data attributes:
    • data-bookmark-id (should be a number like "5")
    • data-id (same as bookmark-id)
    • data-url (the bookmark URL)
    • data-title (the bookmark title)
    • data-parent-id (parent folder ID)
    • data-index (position in folder)
All bookmark elements should have the correct data attributes with actual Chrome bookmark IDs, not synthetic IDs like "bookmark-0".

Test 2: Inter-Folder Drag-Drop Operation

  1. Enable edit mode in FaVault
  2. Find the bookmark "Copilot | Microsoft 365 Copilot" in folder "1"
  3. Drag it to a different folder (e.g., folder "23")
  4. Drop it on the folder
  5. Check the console for any errors
  6. Verify the bookmark appears in the target folder
The bookmark should move successfully without "Bookmark id is invalid" errors. Console should show successful move operation with the correct bookmark ID.

Test 3: Console Log Verification

  1. Open Chrome DevTools Console
  2. Clear the console
  3. Perform an inter-folder drag-drop operation
  4. Look for the drag start log message
